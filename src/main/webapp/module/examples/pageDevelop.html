一、说明<br>
目标页面由html文件，css文件，js文件以及记录设计结果的xml文件组成<br>
其中，html文件及css文件，由美工事先制作完成，js文件及xml文件在开发过程中产生
二、html文件
其实是常规意义上html文件的片段，仅包含原body标记内的部分，不包含html标记、head标记内容，及body标记本身。
常规html:
<html>
<head>
<title>标题</title>
<link href=''>
<link href=''>
<script href=''></script>
<script href=''></script>
</head>
<body>
<div> 页面内容...</div>
</body>
</html>
nim系统的html：
<div> 页面内容...</div>
如果当前页面需要引用同名js,同名css以外的内容，可在html页面中，加入相应的引用：
<link href=''>
<link href=''>
<script href=''></script>
<script href=''></script>
<div> 页面内容...</div>
三、js文件
最好不要在js文件中，直接写浏览器加载后会自动运行的代码，应将javascript代码写在 function函数中 ，由相应的事件调用(如页面的加载事件)。
注：多次加载子页面时，不会重复加载子页面的js文件 四、css文件
五、xml文件
记录设计器的设计内容，主要包括当前页面的标题，是否需要登录等。如果是用于弹出窗口的子页面，还需要定义窗口的高度和宽度。 六、文件加载顺序
通过nim系统访问目标页面时，仅需在地址栏提供html文件的地址，如果在同一目录下存在同名的js,css,xml文件，系统加载html文件后，
会先加载html文件中引用的第三方js和css文件，然后按照css-js-xml的顺序继续加载同名的其它文件（xml文件，会被解析为一系列的javascript代码加载）。
